MRI

Matz's Ruby Interpreter or Ruby MRI is the reference implementation of the Ruby programming language.

Wikipedia
  • Versions:
    • 1.8.7: Will be discontinued in 2013.
    • 1.9.3: Current stable version.
    • 2.0.0-preview1: New features. Comming February 2013.

JRuby

JRuby is a 100% Java implementation of the Ruby programming language. It's Ruby for the JVM.

https://github.com/jruby/jruby/wiki/AboutJRuby
  • Can execute Java inside Ruby.
  • Better threads management.

Rubinius

Provide a rich, high-performance environment for running Ruby code.

Wikipedia
  • Includes a bytecode virtual machine.
  • Compiles to bytecode.
  • Better threads management.

Version manager

  • You can install many interpreters.
  • You can switch the interpreter.
  • You can test your app against various interpreters.
  • Most importants:
    • RVM
    • rbenv

RVM

  • Widely used.
  • Active development.
  • With it you can:
    • Install many versions of ruby.
    • Manage gemsets.
  • It has a GUI for Mac: jewelrybox

RVM

But...

  • It overrides some shell functions (like cd and gem).
  • It install its own packages instead of use the ones available in the system (openssl, ...).
  • Sometimes is a pain.

rbenv

  • Same principles as RVM: manage and control ruby versions.
  • It doesn't install interpreters, only manages them.
  • Configurable via plugins:
    • Install (with ruby-build).
    • Gemsets.
  • Easy. Faster.
  • More UNIX.
  • Only manipulates shell env variables, like PATH.

Gem: Command

  • Part of the ruby interpreter.
  • Package manager for ruby.
  • If you use a manager, each version install its own gems.
  • Basic gems: bundler and rake.
  • It has a config file (~/.gemrc).

Example

Gem: Package

A gem is a packaged Ruby application or library. It has a name (e.g. rake) and a version (e.g. 0.4.16)
  • Once a gem is installed, is available in the ruby load path.

IRB

  • Part of the ruby interpreter.
  • Quick and easy way to execute ruby code.
  • Useful for testing.
  • It can control jobs, manipulate sessions and connect to remote servers.

IRB: Usage example

Bundler: The basics

  • It's a gem.
  • Manage all the dependencies of your app.
  • Resolves the dependency hell.
  • Two key files:
    • Gemfile: ruby file to declare all the needed gems.
    • Gemfile.lock: managed by bundler. Sets all the versions and dependencies. Never write in this file!
  • Install and update gems.
  • All the gems in the Gemfile are available in the app's load path.

Bundler: Gemfile example

Bundler: Ruby usage

Bundler: Tips

  • Always use bundler to manage your app dependencies.
  • Use groups.

  • Use --path and --binstubs options.

  • Use a config file in ~/.bundle/config

Rake

  • It's a gem.
  • The make of ruby.
  • Standard system to create all kind of tasks.
  • Declarative and powerful DSL.
  • Uses a ruby file called Rakefile.

Rake: Usage example

Rails

A extremely basic introduction.

  • It's a meta-gem (a group of gems):

    • actionmailer
    • actionpack
    • activemodel
    • activerecord
    • activesupport
  • It has a complete command line utilities for: code generation, database manipulation, testing, etc.

Rails: Command line basics

  • The rails command:
    • rails console
    • rails server
    • rails generate
    • rails dbconsole
    • rails new app_name
  • rake tasks:
    • Used for common administration tasks.
    • Manipulate the db, assets, etc.
    • You can create your own tasks.
